# AGENTS.md - Instrucciones para Codex en IntegraTronic

## Contexto del proyecto

IntegraTronic es un TFG desarrollado en equipo. El proyecto consiste en un ERP para una tienda física y online de productos informáticos.

Tecnologías previstas:
- Backend: Java + Spring Boot.
- Frontend: React.
- Base de datos: MySQL.
- Arquitectura: API REST y arquitectura por capas.

## Reglas de trabajo

- No trabajar directamente sobre la rama main.
- Antes de modificar archivos, indicar qué archivos se van a tocar y para qué.
- No ejecutar comandos importantes sin explicar antes su finalidad.
- No hacer push sin confirmación del usuario.
- No crear commits sin confirmación del usuario.
- No modificar varios módulos a la vez si no es necesario.
- Priorizar cambios pequeños, claros y revisables.

## Alcance funcional permitido

El proyecto debe mantenerse coherente con las entregas previas del TFG. Los módulos permitidos son:

- Gestión de usuarios y roles.
- Gestión de productos.
- Familias de productos.
- Gestión de stock.
- Movimientos de stock.
- Ventas físicas.
- Pedidos online.
- Clientes / CRM básico.
- Pago online mediante sistema externo.
- Informes y estadísticas.

No añadir módulos nuevos sin confirmación explícita del usuario.

## Forma de responder

Antes de proponer código o cambios:
1. Explicar qué se va a hacer.
2. Indicar qué archivos se tocarían.
3. Explicar el efecto del cambio.
4. Esperar confirmación si el cambio afecta al repositorio.

Mantener las soluciones sencillas y adecuadas para aprobar el TFG.
